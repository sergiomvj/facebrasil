Fontes & Estratégia de Scraping

Seis categorias de fontes mapeadas. APIs oficiais têm prioridade para garantir conformidade legal e estabilidade. Web scraping é estratégia complementar para fontes sem API pública.

Fonte Biblioteca Dados Coletados Tipo
Google Trends pytrends Volume de busca, tópicos relacionados, sazonalidade regional API OFICIAL
Reddit praw Posts hot, subreddits por nicho, engajamento orgânico, perguntas da audiência API OFICIAL
RSS / News Feeds feedparser Publicações de portais de notícias, blogs especializados e concorrentes LIVRE
Redes Sociais Playwright Hashtags emergentes, posts virais, métricas de engajamento público SCRAPING
Dados Governamentais requests + feedparser Estatísticas públicas, boletins oficiais, releases institucionais API OFICIAL
Conteúdo Concorrente Playwright Cobertura editorial de portais similares, temas em destaque, frequência de publicação SCRAPING
scraping_profile_schema · exemplo: revista_gastronomia
// Cada publicação define seu próprio perfil de scraping
tenant_id:       "revista-gastronomia"
sources: {
  google_trends:  { keywords: ["receitas", "restaurantes", "gastronomia"], geo: "BR" },
  reddit:         { subreddits: ["brasil", "gastronomia", "culinaria"], sort: "hot" },
  rss:            { feeds: ["theguardian.com/food", "epicurious.com/feed"] },
  competitors:    { domains: ["veja.abril.com.br/tag/gastronomia"] }
}
schedule: {
  google_trends: "0 */6 * * *",   // a cada 6h
  reddit:        "0 */2 * * *",   // a cada 2h
  rss:           "*/30 * * * *"    // a cada 30min
}
⚖️
Compliance-First — APIs Oficiais como Prioridade
Google Trends e Reddit têm APIs oficiais gratuitas — sempre usar antes de recorrer a scraping. Playwright é reservado para fontes sem alternativa. Scraping é auditado por publicação e requer aprovação antes de ativar em novos domínios.

Stack Tecnológica

Cada camada da plataforma foi escolhida para maximizar velocidade de desenvolvimento, escalabilidade e custo operacional reduzido — especialmente no crescimento de novos tenants.

⚛️
Next.js + React
FRONTEND · APP LAYER
Framework principal com App Router, Server Components e API Routes. Renderização híbrida para performance editorial.
next@14+ react@18+ tailwindcss@3
🗄️
Supabase
DATABASE · AUTH · REALTIME
PostgreSQL gerenciado + Auth + RLS + Realtime + Storage. Backbone de dados da plataforma sem overhead de infraestrutura.
@supabase/supabase-js @supabase/ssr postgres (direto via pgvector)
📈
pytrends + praw
PYTHON · APIS OFICIAIS
Coleta de dados do Google Trends e Reddit via APIs oficiais. Zero risco legal, alta estabilidade, dados confiáveis.
pytrends==4.9.2 praw==7.7.1 feedparser==6.0.11
🎭
Playwright + APScheduler
PYTHON · SCRAPING · JOBS
Browser automation para fontes sem API. APScheduler orquestra todos os jobs de coleta com configuração por tenant.
playwright==1.43+ apscheduler==3.10+ httpx + beautifulsoup4
deployment_options · serviços python
// Opção A — Docker + Railway/Render (recomendado para iniciar)
services:
  scraper-api:
    build: ./python-services
    environment:
      SUPABASE_URL: ${SUPABASE_URL}
      SUPABASE_SERVICE_KEY: ${SUPABASE_SERVICE_KEY}
    deploy: { replicas: 2 }

// Opção B — Celery Workers (para escala de 20+ tenants)
celery:
  broker: "redis://redis:6379/0"
  worker_concurrency: 8
  task_routes:
    scraper.tasks.google_trends: "queue-trends"
    scraper.tasks.reddit:        "queue-social"

Roadmap de Desenvolvimento

Quatro fases sequenciais com entregas incrementais. Valor entregue desde a Fase 1 — cada sprint produz algo utilizável pela redação.

Fase 1 · Semanas 1–2
Fundação — Supabase + Auth + Next.js
Setup do projeto Next.js com App Router. Modelagem do banco Supabase com schema multi-tenant. Implementação das políticas RLS. Sistema de auth com Supabase Auth e seleção de publicação no login.
Entregável: Dashboard vazio mas funcional com auth multi-tenant operando
● Em Desenvolvimento
Fase 2 · Semanas 3–4
Scraping Core — Google Trends + Reddit + RSS
Serviços Python com pytrends, praw e feedparser. Agendamento via APScheduler com configuração por tenant. Pipeline de normalização e armazenamento no Supabase. Primeiras pautas automatizadas chegando ao dashboard.
Entregável: Fluxo completo — scraping → banco → dashboard, para 2 publicações piloto
○ Planejado
Fase 3 · Semanas 5–6
Perfis de Scraping + Interface Editorial
UI para editores configurarem perfis de scraping sem código. Filtros avançados no dashboard por fonte, relevância e data. Sistema de status de pauta (nova, em produção, publicada, descartada). Alertas em tempo real via Supabase Realtime.
Entregável: Editores autônomos — configuram e gerenciam suas próprias pautas
○ Planejado
Fase 4 · Semanas 7–8
Escala, Playwright & Inteligência Editorial
Playwright para redes sociais e fontes sem API. Containerização Docker dos serviços Python. Clustering automático de temas similares. Score de relevância por nicho. Cross-publicação opcional para tendências multi-veículo. Deploy em produção para todos os tenants do grupo.
Entregável: Plataforma completa em produção para todo o grupo de mídia
○ Planejado
