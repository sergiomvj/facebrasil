O que é o FBR-Dev?

Plataforma de gestão do ciclo de vida de software da Facebrasil. Une gestão de issues e sprints (GitHub Projects), comunicação técnica e agentes autônomos que revisam PRs, monitoram deploys e abrem incidentes automaticamente.

🔀

Fragmentação de contexto

Issues no GitHub, discussões no Slack, deploys no Coolify — nenhuma ferramenta fala com a outra. O desenvolvedor perde tempo sincronizando contexto manualmente.

🚨

Alertas sem ação

Sentry detecta o erro, mas ninguém abre o issue. O pipeline quebra às 3h da manhã e ninguém sabe até a manhã seguinte. Reatividade pura.

📋

Revisão de código manual

PRs ficam dias sem review. Checklist de segurança esquecido. Padrões de código inconsistentes entre projetos FBR por falta de automação.

🔭

Visibilidade zero cross-sistema

FBR-Click, FBR-Leads, FBR-Suporte evoluem em silos. Nenhum painel mostra o estado geral de todos os projetos FBR em tempo real.

⚡

Princípio central do FBR-Dev

Todo código nasce de uma conversa no canal de desenvolvimento. Todo deploy gera uma notificação rastreável no FBR-Click. Agentes fecham o loop entre o que foi planejado, o que foi implementado e o que está em produção — sem intervenção humana para as tarefas repetitivas.

Os 5 agentes nativos

Cada agente tem repositório Git próprio, 7 Markdowns de configuração e opera em escopos específicos de repositórios com heartbeat e triggers de webhook definidos.

🔍
Review Bot
Scope: Pull Requests
Checklist automatizado em todo PR aberto: convenções de commit, testes unitários, padrões de segurança e lint. Comenta inline no GitHub.
on: pull_request opened/updated
🚀
Deploy Bot
Scope: CI/CD
Monitora pipelines do GitHub Actions e Coolify. Notifica no FBR-Click e ativa rollback automático em falhas com aprovação do owner.
on: deploy_status change
🚨
Incident Bot
Scope: Sentry + Grafana
Captura erros críticos do Sentry e anomalias do Grafana. Abre issue no GitHub, cria canal de incidente no FBR-Click e notifica on-call.
on: sentry alert / threshold breach
📊
Sprint Bot
Scope: GitHub Projects
Relatórios de sprint semanais com velocity, lead time e DORA metrics. Identifica issues bloqueadas e avisa o dev responsável.
Segunda 9h · sexta 17h
🔐
Security Bot
Scope: Todos os repos FBR
Varredura semanal de dependências (Dependabot API), secrets expostos e vulnerabilidades. Cria PRs de atualização automáticos quando crítico.
Domingo 3h · on: push main

10 camadas de proteção

Segurança integrada desde o schema até a camada de apresentação. Nenhuma camada é opcional — todas verificadas no checklist pré go-live e monitoradas continuamente.

🔐
Auth Humano
iron-session cookie httpOnly, secure, sameSite=lax. Sem localStorage jamais.
🤖
Auth Agente
JWT rotacionado automaticamente a cada 24h via Redis TTL.
🗂️
Row Level Security
RLS em todas as 14 tabelas — isolamento cross-tenant garantido no banco.
🔏
HMAC-SHA256 Webhooks
GitHub, Coolify e Sentry validam assinatura antes de qualquer processamento.
✋
Approval Workflow
Rollback, force push e operações destrutivas exigem aprovação humana prévia.
⏱️
Rate Limiting
60 actions/min por agente · 5 webhook events/min via SlowAPI.
📜
Audit Log Append-Only
Sem UPDATE nem DELETE em agent_action_logs e webhook_events.
🔦
Scope Validation
Agentes validados contra AGENTS.md antes de cada ação. Violação → 403.
🔌
Kill Switch
Admin desconecta qualquer agente em < 5 segundos via FBR-Click.
🧹
Prompt Injection
Sanitização de diff/código antes de enviar ao OpenClaw. SOUL.md carregado primeiro.
