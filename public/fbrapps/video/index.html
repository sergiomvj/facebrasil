00 — Visão do Projeto

Por que este projeto existe

"Uma empresa com presença em múltiplas redes sociais não pode depender de um editor de vídeo humano para cada Short, Reels ou TikTok. O conteúdo de vídeo padronizado deve ser automatizado — com narrativa, identidade de marca e qualidade técnica consistentes."

O FBR-Video é o sistema de automação de produção de vídeo da FBR. Recebe um briefing estruturado, aciona dois agentes OpenClaw especializados que trabalham em sequência — o Roteirista transforma o briefing em um roteiro aprovado pelo Supervisor; o Editor monta e renderiza o vídeo final. O output são arquivos MP4 prontos para publicação nas plataformas de destino.

O sistema segue integralmente a Bíblia FBR v1.0: agentes OpenClaw com 7 Markdowns versionados em Git, orquestração via n8n dedicado, cascata de LLMs com Ollama → Claude → GPT-4o, e toda interação humana centralizada no FBR-Click. Cada agente opera em uma única tarefa por vez — foco total, sem paralelismo dentro do mesmo agente.

02 — Fluxo de Produção

Como um vídeo é processado

Todo trabalho segue um pipeline determinístico em dois estágios com aprovação humana entre eles. O Roteirista cria o roteiro; o Editor transforma em vídeo. O Supervisor aprova cada etapa antes de avançar.

01

Recepção do Briefing

O operador preenche o formulário: plataformas alvo, tipo de vídeo, objetivo, público-alvo, mensagem principal, CTA, tom desejado, narração e trilha preferidas, e faz upload de assets do cliente (vídeos, imagens, áudio).

02

Agente Roteirista — Geração de Roteiro

O agente FBR-Video-Roteirista é acionado pelo n8n. Ele lê o briefing, define a estrutura narrativa adequada à plataforma (Hook → Desenvolvimento → CTA), gera as cenas com narração, direção visual, legendas e lista os assets necessários. LLM primário: Ollama local. Fallback: Claude API → GPT-4o.

03

Aprovação do Roteiro — Supervisor

O roteiro gerado é publicado no canal #fbr-video do FBR-Click. O Supervisor revisa, pode editar cenas individualmente, e aprova ou rejeita com nota. Nenhuma produção começa sem o roteiro aprovado.

04

Agente Editor — Coleta de Assets

O agente FBR-Video-Editor é acionado pelo n8n após aprovação. Ele coleta assets por cena: 1º busca em Pexels Videos, Unsplash, Freesound e Pixabay (gratuitos). Se qualidade insuficiente, gera via RunwayML (b-roll IA) ou ElevenLabs (narração) ou Suno (trilha).

05

Geração de Narração e Trilha (IA)

Se o roteiro requer narração: ElevenLabs gera o áudio usando a voz configurada no brand kit do cliente. Se requer trilha original: Suno API gera música instrumental com o gênero e mood do brand kit. Assets fornecidos pelo cliente têm prioridade.

06

Montagem e Renderização — FFmpeg + Remotion

O Editor monta a timeline: encadeia clipes por cena, sincroniza narração, adiciona trilha em volume reduzido, gera legendas animadas via Remotion e finaliza com marca d'água e identidade visual do brand kit. FFmpeg renderiza o vídeo final. Creatomate atua como fallback de renderização.

07

Variantes por Plataforma

A partir da mesma composição aprovada, o sistema gera automaticamente as variantes nas especificações corretas de cada plataforma selecionada: YouTube Short (9:16), YouTube Video (16:9), Instagram Reels (9:16), TikTok (9:16). Thumbnails gerados para cada formato.

08

Aprovação & Entrega via FBR-Click

O agente posta os vídeos finalizados no canal #fbr-video do FBR-Click como membro ativo — não como webhook anônimo. O Supervisor visualiza, aprova, solicita ajuste ou rejeita. Após aprovação, o operador faz o download ou entrega direta ao cliente.

04 — Roteiro Estruturado

Como os roteiros são organizados

Cada roteiro é um documento JSON estruturado com cenas, narração, direção visual, legendas e lista de assets necessários. O Supervisor revisa e aprova o roteiro antes da produção iniciar.

Princípio central O roteiro é o artefato que separa a fase de intenção da fase de produção. Ele serve de contrato entre o operador (que descreve o que quer) e o editor (que executa). Toda divergência criativa é resolvida no roteiro — antes de gastar tempo e crédito de API na produção.

05 — Agentes OpenClaw FBR

Como os agentes do FBR-Video funcionam

Todo agente do FBR-Video segue o padrão OpenClaw obrigatório da Bíblia FBR: definido por exatamente 7 arquivos Markdown versionados em Git, sem exceção. Cada agente trabalha em uma única tarefa por vez — foco total, sem paralelismo interno.

Princípio: n8n orquestra, OpenClaw executa O n8n (instância dedicada FBR-Video) decide quando e por que um agente é chamado. O OpenClaw Gateway decide o que o agente faz quando chamado. Os dois não se substituem — são camadas complementares e obrigatórias.
Agentes do MVP
📝

FBR-Video-Roteirista

Agente de roteirização. Recebe o briefing, define a estrutura narrativa por plataforma, gera as cenas com narração e direção visual, e lista os assets necessários. Roteiro publicado no FBR-Click para aprovação obrigatória antes da produção.

🎬

FBR-Video-Editor

Agente de produção. Coleta assets (b-roll, imagens, áudio), gera narração via ElevenLabs, trilha via Suno, monta a timeline e renderiza via FFmpeg + Remotion. Entrega as variantes para cada plataforma, com thumbnails inclusos.

Cascata de LLM — padrão FBR
# Cascata LLM — FBR-Video (idêntica ao padrão FBR) Camada 1 — Ollama (Mac Mini M4 32GB via Tailscale) # primário → Uso: roteirização de alto volume, classificação de briefing, seleção semântica → Timeout: 15s · Fallback automático se offline > 30s Camada 2 — Claude API (claude-sonnet-4-6) # fallback 1 → Uso: raciocínio criativo complexo, personalização de narrativa por marca → Timeout: 30s · Fallback se rate limit ou erro Camada 3 — GPT-4o API # contingência → Uso: reserva total · nunca usado como primário → Timeout: 30s · Fallback: alerta crítico no FBR-Click # n8n faz health check nas 3 camadas a cada 60s → publica no Redis # llm.py lê Redis antes de cada chamada — sem latência de health check
06 — Recursos do Sistema

Funcionalidades do MVP

📋

Briefing Multi-Plataforma

Wizard de 4 steps que permite selecionar múltiplas plataformas simultaneamente. Um briefing gera automaticamente variantes para YouTube, Instagram e TikTok.

📝

Roteirização por IA

Agente Roteirista adapta a estrutura narrativa para cada plataforma: Hook 3s → Desenvolvimento → CTA para Shorts; estrutura expandida para vídeos longos. Roteiro revisável cena a cena antes da produção.

🎤

Narração com ElevenLabs

Voz personalizada por cliente no brand kit. Narração gerada automaticamente a partir do texto do roteiro. Múltiplas vozes disponíveis com preview antes de selecionar.

🎵

Trilha Sonora com Suno

Música instrumental original gerada por IA no gênero e mood do brand kit. Sem problemas de direitos autorais. Loops de 30-60s ajustados automaticamente à duração do vídeo.

🎥

B-roll em Cascata

1º busca em Pexels Videos (gratuito e sem copyright). Se não atender, gera footage original via RunwayML. Assets fornecidos pelo cliente têm prioridade absoluta.

⚡

Renderização FFmpeg

Motor de renderização nativo, sem custos de API por vídeo. Creatomate como fallback de alta confiabilidade. Specs corretas para cada plataforma geradas automaticamente.

💬

Legendas Automáticas

Remotion gera legendas animadas sincronizadas com a narração. Estilo configurável no brand kit: posição, fonte, cor de fundo, animação de entrada.

✅

Aprovação Dupla via FBR-Click

O Supervisor aprova primeiro o roteiro e depois o vídeo final. Dois checkpoints humanos obrigatórios. O agente posta no canal #fbr-video como membro ativo — nunca como webhook anônimo.

🏢

Brand Kit de Vídeo

Cada cliente tem voz ElevenLabs, gênero musical, estilo de transição, posição e estilo de legenda, e marca d'água configurados. Tudo aplicado automaticamente em cada produção.

📊

Dashboard & Analytics

Métricas de produção: vídeos por plataforma, tempo médio de produção, taxa de aprovação de roteiros, distribuição de uso por serviço de IA.

07 — Roadmap

Fases de desenvolvimento

O MVP é dividido em três frentes alinhadas ao processo obrigatório de criação de agentes FBR: fundação de infraestrutura, configuração dos agentes OpenClaw com seus 7 Markdowns, e interface + entrega via FBR-Click.

Fase 1
Fundação

Infraestrutura & Backend

  • VPS Hostinger + Docker Compose (8 vCPU / 16GB)
  • FastAPI + PostgreSQL 16 + Redis 7
  • n8n instância dedicada FBR-Video
  • MinIO storage configurado (3 buckets)
  • Tailscale para acesso ao Ollama
  • Nginx + Certbot + Grafana
  • Modelo de dados criado (jobs, roteiros, assets)
  • Container FFmpeg worker configurado
  • Integrações de API: ElevenLabs, Suno, Pexels
Fase 2
Agentes

OpenClaw & Pipeline

  • 7 Markdowns do Roteirista (PR obrigatório)
  • 7 Markdowns do Editor (PR obrigatório)
  • Registro dos agentes no FBR-Click
  • Período de observação (7 dias)
  • Cascata LLM: Ollama → Claude → GPT-4o
  • Pipeline de roteirização por plataforma
  • Coleta de assets (Pexels → RunwayML)
  • Geração de narração (ElevenLabs)
  • Geração de trilha (Suno)
  • Renderização FFmpeg + Remotion
  • Geração de variantes por plataforma
  • Brand kit de vídeo engine
Fase 3
Interface

UI & FBR-Click

  • Wizard de briefing multi-plataforma (Next.js)
  • Revisor de roteiro cena a cena
  • Player de vídeo com tabs por plataforma
  • Canal #fbr-video no FBR-Click
  • Agentes como membros ativos
  • Dashboard de métricas e analytics
  • Banco de referências de estilo
  • Brand kits com configuração de vídeo
  • Aprovação formal de operação plena
