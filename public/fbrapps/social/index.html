01

O que é o FBR-Social

O FBR-Social é o sistema de automação de artes visuais para redes sociais do Facebrasil Group. Enquanto o FBR-Design cobre a produção gráfica geral — cartões, banners, flyers, apresentações — o FBR-Social é especializado exclusivamente em formatos de redes sociais: Feed, Stories, Reels, Covers, Thumbnails, Carrosséis e demais formatos de cada plataforma.

Cada produto do Facebrasil Group possui identidade visual própria e presença em múltiplas redes sociais. O FBR-Social centraliza a produção desses materiais, garantindo consistência de marca, velocidade de entrega e qualidade profissional — sem depender de ferramentas externas de design ou de designers manuais para cada peça.

Princípio fundamental: O FBR-Social não inventa — ele executa. Os assets base (logos, fontes, paletas de cores) vêm do FBR-Design via brand kit. O FBR-Social aplica esses assets com maestria nos formatos e dimensões corretos de cada rede social.
🎯

Foco Total em Social

Especializado em formatos de redes sociais. Conhece as especificações exatas de cada plataforma e formato — dimensões, safe zones, limites de texto, aspect ratios.

🔗

Integrado ao FBR-Design

Consome brand kits do FBR-Design via API. Logos, fontes e paletas são fornecidos pelo sistema irmão — sem duplicação de dados, sem inconsistências de marca.

🏢

Multi-Produto FBR

Atende todos os produtos do Facebrasil Group. Cada produto tem seu perfil de brand kit, redes ativas e templates específicos. Um sistema, múltiplas marcas.

⚡

Variantes Automáticas

A partir de um único briefing, gera automaticamente todas as variantes de formato para todas as redes ativas do produto — Feed, Stories, LinkedIn, TikTok e mais.

A relação entre os sistemas

FBR-Design é a fonte de verdade das identidades visuais. FBR-Social é o executor especializado em social media. Quando um brand kit é atualizado no FBR-Design, o FBR-Social reflete automaticamente essa atualização em todos os jobs subsequentes. Os sistemas se comunicam via REST API autenticada — nunca compartilham banco de dados.

04

Pipeline de Produção

O pipeline do FBR-Social segue a mesma lógica robusta do FBR-Design, com adaptações específicas para redes sociais: busca de brand kit no sistema irmão, seleção de template por rede/formato e geração automática de variantes.

01
Recebimento do Briefing

Operador submete briefing via frontend: produto FBR, redes alvo, tipo de conteúdo (lançamento, promoção, informativo, institucional…), textos, referências visuais e tom desejado. n8n aciona o agente via webhook.

02
Busca de Brand Kit no FBR-Design

Agente consulta FBR-Design API para recuperar o brand kit do produto FBR solicitado: logos (claro/escuro), paleta de cores, fontes, guidelines de uso. Se brand kit local estiver em cache e atualizado, usa cache. Sincronização automática a cada 24h.

03
Seleção de Templates por Rede

Para cada rede social ativa do produto, o agente seleciona o template mais adequado via busca semântica: tipo de conteúdo, tom, estilo visual, setor. Retorna top 3 candidatos por rede com score de compatibilidade.

04
Busca de Assets Visuais

FBR-Social-AssetFinder (agente dedicado) busca imagens complementares em Unsplash, Pexels e Pixabay com base na descrição semântica do briefing. Para imagens premium, aciona MidJourney com aprovação humana obrigatória.

05
Composição HTML por Variante

Para cada rede/formato ativo, o agente injeta variáveis no template HTML: textos, logo do produto, cores da paleta, imagens dos assets. Dimensões fixas por formato garantem corte correto nas plataformas.

06
Renderização das Variantes

HTMLCSStoImage API renderiza cada variante nas dimensões corretas. Fallback automático para Bannerbear API. Outputs salvos no MinIO com nomenclatura padronizada por produto, rede e formato.

07
Aprovação no FBR-Click

Agente posta no canal #fbr-social como membro ativo. Preview de todas as variantes geradas, organizadas por rede social. Tag @responsavel-produto para aprovação. Supervisor pode aprovar individualmente por variante ou aprovar tudo de uma vez.

08
Entrega do Pacote

Após aprovação, pacote ZIP disponível para download com todas as variantes aprovadas, organizadas em pastas por rede social. Nomenclatura clara: produto_rede_formato_data.png

Princípios de isolamento

FBR-Social roda em VPS Hostinger isolada — falha não cascadeia para FBR-Design ou outros sistemas FBR. A comunicação com o FBR-Design ocorre exclusivamente via REST API autenticada. Proibido acesso direto a banco de dados cruzado. Frontend nunca acessa o backend diretamente — todo request passa pelo proxy Next.js.

Cascata de LLM (idêntica ao FBR-Design)

Camada 1 — Ollama (Mac Mini M4 32GB via Tailscale)
  → Uso: classificação de briefing, scoring de templates, seleção por rede social
  → Timeout: 15s · Fallback: se offline > 30s → Camada 2

Camada 2 — Claude API (claude-sonnet-4-6)
  → Uso: raciocínio complexo, composição de variáveis, adaptação por plataforma
  → Timeout: 30s · Fallback: se rate limit ou erro → Camada 3

Camada 3 — GPT-4o API (contingência)
  → Uso: reserva total · NUNCA primário
  → Fallback: alerta crítico no FBR-Click · owner notificado

Estrutura Multi-Produto

Cada produto do Facebrasil Group é cadastrado no FBR-Social com seu perfil completo: redes sociais ativas, brand kit vinculado ao FBR-Design, responsável de aprovação e configurações específicas de estilo por plataforma. 

Tabela: fbr_products
  id              UUID
  nome            string          -- ex: "FBR-Design", "FBR-Click", "FBR-Social"
  slug            string          -- ex: "fbr-design"
  brand_kit_id    UUID            -- ID no FBR-Design (sync via BrandSync)
  redes_ativas    string[]        -- ["instagram", "facebook", "linkedin", ...]
  responsavel_id  UUID → users    -- Owner para aprovação no FBR-Click
  ativo           boolean
  criado_em       timestamp
Extensibilidade: Quando um novo produto FBR for lançado, basta cadastrá-lo na tabela fbr_products, vinculá-lo a um brand kit no FBR-Design, e configurar as redes ativas. O sistema já está pronto para atendê-lo.

Princípios Absolutos

O FBR-Social herda todos os princípios da Bíblia FBR e adiciona os específicos para produção de social media.

🔒 Isolamento Total

VPS dedicada. Falha no FBR-Social não afeta FBR-Design nem qualquer outro sistema FBR. Comunicação apenas via REST API autenticada.

✋ Aprovação Humana

Todo material para uso externo passa por aprovação humana no FBR-Click. Agentes nunca entregam diretamente para as redes sociais. Publicação é responsabilidade humana.

📐 Dimensões Corretas

Zero tolerância para artes com dimensões erradas. Cada formato tem especificação rígida. Sistema recusa renderização fora das especificações de cada rede.

🎨 Brand Kit Inviolável

Logos, cores e fontes do brand kit não são modificados sem aprovação explícita do responsável pelo produto. Consistência de marca é inegociável.

📜 Licenças de Imagem

NUNCA usar imagens sem licença para uso comercial digital. Assets free de Unsplash/Pexels/Pixabay são verificados. MidJourney sempre requer aprovação humana.

🔄 Sincronização Confiável

Brand kits sincronizados do FBR-Design a cada 24h e via webhook em atualizações. Cache local com versionamento. Nunca usar brand kit desatualizado silenciosamente.

Filosofia OpenClaw — nunca negociável

Todo agente do FBR-Social é definido por exatamente 7 arquivos Markdown versionados em Git. Proibido CrewAI, AutoGen, LangGraph ou qualquer framework externo de agentes. n8n decide quando e por que um agente é chamado. OpenClaw decide o que o agente faz. Cada agente trabalha em uma única tarefa por vez — sem paralelismo interno. Todo agente tem owner humano designado e kill switch funcional.
